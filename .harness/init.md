# init.md - Bootstrap Contract

Run this before starting work.

## 1. Read

- `.harness/AGENTS.md`
- `.harness/README.md`
- `.harness/progress.md`
- `.harness/session-handoff.md`
- `.harness/feature-list.json`
- relevant files in `.harness/docs/`

## 2. Confirm Scope

Before implementation, identify current feature, success criteria, likely changed files, verification commands, and risks.

If the task is ambiguous, ask a short clarifying question or state the assumption.

## 3. WIP Rule

WIP = 1. Do not start a second feature while one is active.

## 4. Work Mode

Use Planner -> Implementer -> Verifier.

## 5. Closeout

Update `.harness/progress.md` and `.harness/session-handoff.md`. Record architectural decisions in Decisions.
